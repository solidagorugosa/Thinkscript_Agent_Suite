# thinkscript_bot

A prompt-orchestrated ThinkScript Copilot framework that uses agents, instructions, prompts, and reusable skills to support **code generation**, **explanation of code**, **debugging**, **validation**, and **strategy conversion** in the Thinkscript language.

## Overview

This repository provides a modular LLM assistance layer for ThinkScript authoring. It is organized so behavior, task routing, and domain knowledge remain separated and maintainable.

The project follows an **Agent + Skills + Prompt Pack** pattern:

- **Agent definitions** establish role and behavioral boundaries.
- **Instructions** enforce consistent authoring rules and constraints.
- **Prompt templates** support task-specific workflows.
- **Skills** provide reusable ThinkScript-specific guidance and reference material.

## Capabilities

This framework is designed to assist with:

- Generating ThinkScript studies and indicators
- Explaining existing ThinkScript code
- Debugging syntax and logic issues
- Validating scripts against constraints without rewriting them
- Converting plain-English trading rules into ThinkScript strategy structure

## Architecture

This project uses a **prompt-orchestrated LLM assistant architecture** for ThinkScript development.

### Component Roles

- **Agent**
  - Routes ThinkScript tasks and applies role-specific behavior.
- **Instructions**
  - Define syntax, safety, and formatting expectations.
- **Prompts**
  - Provide structured workflows for generate, explain, debug, validate, and strategy conversion tasks.
- **Skills**
  - Supply reusable ThinkScript-specific guidance and reference material.

## How It Works

1. A ThinkScript task is routed to the ThinkScript agent.
2. The agent applies shared authoring instructions.
3. A task-specific prompt is selected.
4. The ThinkScript skill supplies domain guidance and references.
5. The agent self-validates output using the deterministic parser when terminal execution is available.
6. The model returns structured output for iteration or review.

## Output Enforcement

This repository includes a deterministic hook policy at `docs/hook-policy.md`.

That policy defines:
- prompt-specific output contracts
- minimal ThinkScript structural checks
- self-validation rules applied by the agent before returning code

## Deterministic Parser

The repo includes a local ThinkScript parser/linter pipeline:

- `scripts/build_thinkscript_manifest.py` — builds a function/constant manifest from local docs
- `scripts/thinkscript_parser.py` — parses and lints ThinkScript code
- `scripts/self-check-thinkscript.ps1` — runs manifest build and parser in one step
- `scripts/validate-thinkscript-output.ps1` — validates full output files with code block extraction

When the agent has terminal access (agent mode), it runs these automatically before returning code. When terminal access is unavailable, the agent applies equivalent rules as an inline self-validation checklist.

No end-user action is required.
